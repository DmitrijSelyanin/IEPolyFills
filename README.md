this is some open-source polyfills to implement modern web standards in Internet Explorer. let.js polyfills for let by replacing it with var. this polyfill requires ES5 and later support, so IE10 or IE11 work fine with it but IE9 and older don't. const.js polyfills for const by replacing it with var. this polyfill requires ES5 and later support, so IE10 or IE11 work fine with it but IE9 and older don't. arrow.js polyfills for arrow functions (() => {}) by replacing them with normal functions. this polyfill requires ES5 and later support, so IE10 or IE11 work fine with it but IE9 and older don't. class.js polyfills for classes using a complex mess that acts as an ES5-based class runtime of sorts. developer eyes will go pop when they see this. requires IE10 or newer because of IE10 being the first IE to have ES5 JS.
